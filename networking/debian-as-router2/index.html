使用Debian路由及拨号2

发表于 2021-08-31 更新于 2023-03-11 分类于 networking

背景

续上一篇将Debian配置成路由器，本篇讲介绍如何在Debian上使用同样的软件实现 openwrt 上的 helloword 插件功能。

声明

本文章仅为记录自己在学习openwrt中的helloworld插件以更深入的了解linux网络以及iptables时的浅薄理解！所有的配置均可在github上找到。

  • 准备工作

    首先我们安装用到的软件。大部分软件都是在上一篇当中用过的。

    • pdnsd

      根据Debian官网显示，由于很久无人维护的原因已经从官方的软件源当中移除了。好在我们还可以从debian的官网下载到pdnsd的发行包以及文档。

      下载好发行包之后使用下面命令进行安装

      dpkg -i pdnsd_1.2.9a-par-2_amd64.deb

      配置文件位于 /etc/pdnsd.conf，首先我们备份这个然后重新创建一个后面进行手动配置

    • iptables

      系统自带

    • dnsmasq

      上篇已经安装，请自行参考

    • ipset

      系统自带

      xray的github以及文档

      根据github的教程使用下面命令进行安装：

      安装的目录位于 */usr/local/etc/xray/*，配置文件的目录 /usr/local/etc/xray/config.json

  • 原理

    尝试先用文字进行描述吧，后面可能会有一个流程图也不一定

    1. 预定配置好ipset中用到的一些集合：china, gfwlist, whitelist, blacklist
    2. xray 作为客户端与服务端进行连接，使用tcp协议。
    3. dnsmasq 可以读取写入ipset，iptables可以根据ipset来设置规则
    4. pdnsd 可以对解析结果进行缓存，也可以设置为仅用tcp协议进行解析，这个可以完美地与xray搭配。把gfwlist中的域名由pdnsd通过xray转发到pdnsd中设置的上游dns服务器进行解析。
    5. dnsmasq 提供本地dns解析服务，dnsmasq中可以配置规则，将指定的域名解析回来的ip地址加入到指定的ipset中并进行缓存。dnsmasq还可以配置指定的域名使用指定的dns服务进行解析，这样可以让gfwlist中的域名强制用pdnsd进行解析。
    6. iptables 设置规则如果在China/whitelist集合中则直接转发，如果是在gfwlist/blacklist集合中则转发给xray所监听的端口，然后由xray转发到服务器，服务器再转发到pdnsd指定的上游dns服务器进行解析然后返回结果。

    重要提示

    整个流程看起来似乎不是那么复杂，但是需要了解我们在上网的时候涉及到dns解析，以及目标ip无法访问的情况。刚开始的时候我尝试的时候所有的工具都按照预定的方式在工作，但是唯独pdnsd进行解析的时候永远也解析不到结果。那是因为pdnsd使用的上游服务器为 8.8.8.8/8.8.4.4，这两个ip本来就是无法访问的。因此这两个ip要手动加入到gfwlist中，这样由pdnsd发往 8.8.8.8/8.8.4.4 的dns解析请求才会根据iptables的规则由xray进行转发；而gfwlist集合中的其他ip地址是根据dnsmasq的设置在域名解析请求之后才会被添加到gfwlist集合中然后缓存的。

      • whitelist: 这个集合简单很多，只需要将不在 china 集合中，也不想走 xray 的域名加入到其中就可以了。

        此处我们必须要将 xray 中配置的服务器域名加入到这个集合中，此处我们不必手动添加，可以在初次访问的时候由 dnsmasq 自动添加进去
